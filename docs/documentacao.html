    Introdução à Linguagem Charcot

    Bem-vindo à documentação oficial da linguagem Charcot, a primeira linguagem de programação projetada especificamente para o desenvolvimento de aplicações médicas. Esta documentação serve como guia completo para aprender e utilizar a linguagem Charcot de forma eficiente.

    Nota: Charcot está atualmente na versão 0.1.0 (Preview). Algumas características podem mudar antes da versão estável 1.0.

    Por que Charcot?

    O desenvolvimento de software médico é um campo com desafios únicos. Os sistemas de saúde exigem alto nível de segurança, conformidade regulatória, integração com padrões médicos e capacidade de modelar fluxos de trabalho complexos do ambiente hospitalar. Embora linguagens de programação tradicionais possam ser usadas para desenvolver aplicações médicas, nenhuma delas foi projetada especificamente para este propósito.

    Charcot preenche esta lacuna oferecendo:

    • Segurança no núcleo da linguagem: Detecta erros relacionados à medicina em tempo de compilação
    • Sintaxe intuitiva: Familiar a desenvolvedores Python, mas com os benefícios do Rust
    • Padrões médicos nativos: FHIR, HL7, SNOMED CT e outros integrados à biblioteca padrão
    • Primitivas específicas: Tipos e funções para workflows hospitalares comuns
    • Segurança e auditoria: Rastreabilidade de todas as operações para compliance regulatória

    Um Exemplo Inicial

    Vamos ver um exemplo básico de código Charcot que calcula o Índice de Massa Corporal (IMC) de um paciente:

    calculadora_imc.chc
    // Calculadora de IMC em Charcot
usar paciente::dados_antropometricos;

// Função que calcula o IMC
func calcular_imc(peso_kg: Decimal, altura_m: Decimal) -> Resultado<Decimal> {
    // Validação de entrada
    se peso_kg <= 0.0 {
        retornar Erro("Peso deve ser maior que zero");
    }
    
    se altura_m <= 0.0 {
        retornar Erro("Altura deve ser maior que zero");
    }
    
    // Cálculo do IMC: peso (kg) / altura² (m)
    let imc = peso_kg / (altura_m * altura_m);
    
    // Arredondar para uma casa decimal
    Ok(arredondar(imc, 1))
}

// Função para classificar o IMC
func classificar_imc(imc: Decimal) -> String {
    match imc {
        < 18.5 => "Baixo peso",
        < 25.0 => "Peso normal",
        < 30.0 => "Sobrepeso",
        < 35.0 => "Obesidade grau I",
        < 40.0 => "Obesidade grau II",
        _ => "Obesidade grau III",
    }
}

// Função principal
func principal() {
    let paciente = Paciente::buscar("12345");
    
    // Usar tipos com unidades
    let peso = paciente.peso(); // Tipo: Peso (kg)
    let altura = paciente.altura(); // Tipo: Altura (m)
    
    match calcular_imc(peso.valor(), altura.valor()) {
        Ok(imc) => {
            let classificacao = classificar_imc(imc);
            imprimir(f"Paciente: {paciente.nome_completo()}");
            imprimir(f"IMC: {imc} - Classificação: {classificacao}");
        },
        Erro(e) => {
            imprimir(f"Erro ao calcular IMC: {e}");
        }
    }
}

    Este simples exemplo já demonstra várias características importantes de Charcot:

    • Verificação de entradas: Validação explícita de entradas para prevenir cálculos inválidos
    • Tratamento de erros: Uso do tipo Resultado para representar operações que podem falhar
    • Pattern matching: Utilização de match para classificação concisa
    • Interpolação de strings: Formatação de strings com valores de variáveis
    • Integração com conceitos médicos: Acesso a dados de pacientes de forma natural

    Em versões mais avançadas deste exemplo, poderíamos utilizar tipos específicos da medicina com validação integrada, como:

    calculadora_imc_avancada.chc
    usar paciente::dados_antropometricos;
usar medidas::antropometria;

// Tipos com verificação integrada
tipo Peso = Decimal<0.1..635.0, "kg">; // Intervalo válido com unidade
tipo Altura = Decimal<0.3..2.5, "m">;
tipo IMC = Decimal<10.0..70.0, "kg/m²">;

// Enumeração para classificação
enum ClassificacaoIMC {
    BaixoPeso,
    PesoNormal,
    Sobrepeso,
    Obesidade1,
    Obesidade2,
    Obesidade3,
}

// Função com tipos validados
func calcular_imc(peso: Peso, altura: Altura) -> IMC {
    // Não é mais necessário validar entradas - o sistema de tipos garante valores válidos
    let imc_valor = peso.valor() / (altura.valor() * altura.valor());
    IMC::novo(arredondar(imc_valor, 1))
}

// Classificação com segurança de tipos
func classificar_imc(imc: IMC) -> ClassificacaoIMC {
    match imc.valor() {
        < 18.5 => ClassificacaoIMC::BaixoPeso,
        < 25.0 => ClassificacaoIMC::PesoNormal,
        < 30.0 => ClassificacaoIMC::Sobrepeso,
        < 35.0 => ClassificacaoIMC::Obesidade1,
        < 40.0 => ClassificacaoIMC::Obesidade2,
        _ => ClassificacaoIMC::Obesidade3,
    }
}

    Para quem é Charcot?

    Charcot foi criada pensando em diversos perfis de usuários:

    • Desenvolvedores que trabalham em software médico
    • Profissionais de saúde com experiência em programação
    • Estudantes de informática médica e bioinformática
    • Equipes de TI em hospitais e clínicas
    • Empresas que desenvolvem sistemas de saúde

    Filosofia de Design

    Charcot é guiada pelos seguintes princípios:

    1. Segurança acima de tudo - Prevenir erros em tempo de compilação é sempre preferível à detecção em tempo de execução
    2. Expressividade do domínio - O código deve refletir naturalmente conceitos médicos
    3. Verificabilidade - O comportamento do programa deve ser verificável e auditável
    4. Performance - Em ambientes críticos, a eficiência importa
    5. Interoperabilidade - Integração nativa com padrões médicos estabelecidos
    Importante: Charcot não pretende substituir sistemas de apoio à decisão clínica ou diagnósticos médicos. É uma ferramenta para desenvolver software que auxilia profissionais de saúde, mas a decisão final sempre deve ser tomada por um profissional qualificado.

    Casos de Uso Principais

    Charcot é especialmente adequada para:

    Tipo de Aplicação Características Benefícios da Linguagem
    Sistemas de Prescrição Eletrônica Gerenciamento de prescrições médicas, verificação de doses e interações Verificação estática de interações medicamentosas, validação de dosagens
    Prontuário Eletrônico (EHR) Armazenamento e gestão de informações clínicas de pacientes Integração com FHIR, verificação de permissões, auditoria
    Sistemas de Apoio à Decisão Clínica Auxiliar diagnósticos e tratamentos baseados em evidências Modelagem de protocolos clínicos, integração com terminologias médicas
    Monitoramento de Pacientes Acompanhamento de sinais vitais e alertas em tempo real Programação assíncrona segura, tipos para sinais vitais com validação
    Gestão Hospitalar Gerenciamento de leitos, internações e alta de pacientes Primitivas para workflows hospitalares, transações atômicas
